What is DNS?¿Qué es DNS?

The Domain Name System (DNS) is the internet’s address book. It translates human-readable domain names like www.example.com into machine-readable IP addresses like 93.184.216.34.El Sistema de Nombres de Dominio (DNS) es la guía telefónica de internet. Traduce nombres de dominio legibles como www.example.com a direcciones IP como 93.184.216.34.

Why DNS MattersPor qué DNS es importante

Without DNS, users would have to memorize numeric IP addresses for every website they visit.Sin DNS, los usuarios tendrían que memorizar direcciones IP numéricas para cada sitio web.

DNS is a distributed, hierarchical database that handles billions of queries daily. It is one of the most critical pieces of internet infrastructure — a single point of failure in DNS can make entire regions of the internet unreachable.DNS es una base de datos distribuida y jerárquica que maneja miles de millones de consultas diariamente. Es una de las piezas más críticas de la infraestructura de internet — un fallo en DNS puede hacer que regiones enteras de internet sean inaccesibles.

Key DNS ServicesServicios Clave de DNS

Host-to-IP MappingMapeo Host a IP

Translates domain names to IPv4 (A record) or IPv6 (AAAA record) addresses.Traduce nombres de dominio a direcciones IPv4 (registro A) o IPv6 (registro AAAA).

This is the primary function of DNS. When you type a URL in your browser, DNS resolves it to the server’s IP address so the connection can be established.Esta es la función principal de DNS. Cuando escribes una URL en tu navegador, DNS la resuelve a la dirección IP del servidor para establecer la conexión.

Host Aliasing (CNAME)Alias de Host (CNAME)

Allows one domain name to point to another, enabling canonical name mappings.Permite que un nombre de dominio apunte a otro, habilitando mapeos de nombres canónicos.

Example: www.example.com → example.com. CNAME records are commonly used for subdomains, CDN integration, and service migration without changing public-facing URLs.Ejemplo: www.example.com → example.com. Los registros CNAME se usan comúnmente para subdominios, integración CDN y migración de servicios sin cambiar URLs públicas.

Email Routing (MX)Enrutamiento de Email (MX)

MX records direct email to the correct mail servers for a domain.Los registros MX dirigen el correo a los servidores correctos de un dominio.

MX records include a priority value, allowing failover: if the primary mail server is down, email is routed to a backup server with a higher priority number.Los registros MX incluyen un valor de prioridad, permitiendo failover: si el servidor principal está caído, el correo se envía al servidor de respaldo con mayor número de prioridad.

Load BalancingBalanceo de Carga

DNS can return multiple IP addresses for a single domain, distributing traffic via round-robin.DNS puede devolver múltiples IPs para un dominio, distribuyendo tráfico mediante round-robin.

Large-scale services like Google and Amazon use DNS-based load balancing to distribute users across multiple data centers worldwide. This is the simplest form of global load distribution.Servicios a gran escala como Google y Amazon usan balanceo de carga basado en DNS para distribuir usuarios entre múltiples centros de datos. Esta es la forma más simple de distribución global de carga.

DNS HierarchyJerarquía DNS

DNS is organized as a hierarchical tree with three main levels, from the root down to individual domains.DNS está organizado como un árbol jerárquico con tres niveles principales, desde la raíz hasta dominios individuales.

DNS Server TypesTipos de Servidores DNS

Root ServersServidores Raíz

13 logical servers (A through M), distributed across ~1,000 physical instances worldwide via anycast.13 servidores lógicos (A a M), distribuidos en ~1,000 instancias físicas en todo el mundo mediante anycast.

Root servers are managed by 12 independent organizations (ICANN, NASA, U.S. Army, etc.). They don’t know the final IP — they only point to the correct TLD server. They are critical infrastructure protected by massive DDoS mitigation.Los servidores raíz son administrados por 12 organizaciones independientes (ICANN, NASA, Ejército de EE.UU., etc.). No conocen la IP final — solo apuntan al servidor TLD correcto. Son infraestructura crítica protegida con mitigación masiva de DDoS.

TLD ServersServidores TLD

Manage top-level domains: .com, .org, .net, .edu, country codes (.mx, .uk, .jp).Administran dominios de nivel superior: .com, .org, .net, .edu, códigos de país (.mx, .uk, .jp).

Verisign manages .com and .net TLDs, handling over 170 billion queries daily. Each TLD server knows which authoritative name servers are responsible for domains under that TLD.Verisign administra los TLDs .com y .net, manejando más de 170 mil millones de consultas diarias. Cada servidor TLD sabe qué servidores autoritativos son responsables de los dominios bajo ese TLD.

Authoritative ServersServidores Autoritativos

Hold the actual DNS records for a domain. They provide the definitive answer to DNS queries.Contienen los registros DNS reales de un dominio. Proporcionan la respuesta definitiva a consultas DNS.

Organizations either run their own authoritative DNS (BIND9, PowerDNS) or use managed services (Route 53, Cloudflare DNS). These servers are the source of truth for a domain’s records.Las organizaciones ejecutan su propio DNS autoritativo (BIND9, PowerDNS) o usan servicios administrados (Route 53, Cloudflare DNS). Estos servidores son la fuente de verdad para los registros de un dominio.

Recursive ResolversResolvers Recursivos

Act as intermediaries that perform the full DNS lookup on behalf of clients (e.g., 8.8.8.8, 1.1.1.1).Actúan como intermediarios que realizan la búsqueda DNS completa en nombre de los clientes (ej., 8.8.8.8, 1.1.1.1).

Recursive resolvers cache responses to reduce latency and load on authoritative servers. Your ISP provides one by default, but public resolvers like Google (8.8.8.8) and Cloudflare (1.1.1.1) are popular alternatives.Los resolvers recursivos almacenan respuestas en caché para reducir latencia y carga en servidores autoritativos. Tu ISP proporciona uno por defecto, pero resolvers públicos como Google (8.8.8.8) y Cloudflare (1.1.1.1) son alternativas populares.

DNS Resolution FlowFlujo de Resolución DNS

When a client queries a domain name, the recursive resolver walks the DNS hierarchy to find the IP address.Cuando un cliente consulta un nombre de dominio, el resolver recursivo recorre la jerarquía DNS para encontrar la dirección IP.

DNS Caching & TTLCaché DNS y TTL

DNS responses are cached at multiple levels to reduce lookup latency. Each record has a Time-To-Live (TTL) that controls how long it stays cached.Las respuestas DNS se almacenan en caché en múltiples niveles para reducir la latencia. Cada registro tiene un Time-To-Live (TTL) que controla cuánto tiempo permanece en caché.

Browser CacheCaché del Navegador

First layer checked. Chrome caches DNS for up to 60 seconds by default.Primera capa verificada. Chrome almacena DNS en caché hasta 60 segundos por defecto.

View Chrome’s DNS cache at chrome://net-internals/#dns. Browsers respect the TTL from DNS responses, but may apply their own minimum cache duration.Ver la caché DNS de Chrome en chrome://net-internals/#dns. Los navegadores respetan el TTL de las respuestas DNS, pero pueden aplicar su propia duración mínima de caché.

OS CacheCaché del SO

Operating system resolver cache (e.g., systemd-resolved, dnsmasq).Caché del resolver del sistema operativo (ej., systemd-resolved, dnsmasq).

The OS also checks /etc/hosts before querying DNS. On macOS, use sudo dscacheutil -flushcache to clear. On Windows, use ipconfig /flushdns.El SO también revisa /etc/hosts antes de consultar DNS. En macOS, usa sudo dscacheutil -flushcache para limpiar. En Windows, usa ipconfig /flushdns.

Resolver CacheCaché del Resolver

Recursive resolvers (ISP, 8.8.8.8) cache responses per TTL, serving millions of users.Los resolvers recursivos (ISP, 8.8.8.8) almacenan respuestas según el TTL, sirviendo a millones de usuarios.

A popular domain cached at a major resolver can save billions of queries to authoritative servers. Low TTLs (e.g., 60s) enable fast failover but increase query volume. High TTLs (e.g., 86400s) reduce load but delay propagation of changes.Un dominio popular en caché de un resolver importante puede ahorrar miles de millones de consultas. TTLs bajos (ej., 60s) permiten failover rápido pero aumentan el volumen de consultas. TTLs altos (ej., 86400s) reducen carga pero retrasan la propagación de cambios.

TTL StrategyEstrategia de TTL

Choosing the right TTL is a trade-off between performance and flexibility.Elegir el TTL correcto es un balance entre rendimiento y flexibilidad.

Before a migration, lower TTL to 60s a few days in advance. After the migration is stable, raise TTL back to 3600s or higher. This pattern minimizes downtime during DNS record changes.Antes de una migración, reduce el TTL a 60s unos días antes. Después de que la migración es estable, sube el TTL a 3600s o más. Este patrón minimiza el tiempo de inactividad durante cambios de registros DNS.

DNS SecuritySeguridad DNS

DNS was designed without security in mind. Several threats and defenses have emerged over time.DNS fue diseñado sin seguridad en mente. Han surgido varias amenazas y defensas con el tiempo.

DNSSEC (DNS Security Extensions)DNSSEC (Extensiones de Seguridad DNS)

Adds cryptographic signatures to DNS responses, allowing resolvers to verify that records have not been tampered with.Agrega firmas criptográficas a las respuestas DNS, permitiendo a los resolvers verificar que los registros no han sido alterados.

DNSSEC uses a chain of trust from the root zone down to individual records. It adds RRSIG, DNSKEY, DS, and NSEC record types. It does NOT encrypt queries — for that, use DNS-over-HTTPS (DoH) or DNS-over-TLS (DoT).DNSSEC usa una cadena de confianza desde la zona raíz hasta registros individuales. Agrega tipos de registro RRSIG, DNSKEY, DS y NSEC. NO cifra las consultas — para eso, usa DNS-over-HTTPS (DoH) o DNS-over-TLS (DoT).

DDoS Attacks on DNSAtaques DDoS a DNS

Attackers overwhelm DNS servers with traffic, making domains unreachable (e.g., 2016 Dyn attack took down Twitter, Reddit, Netflix).Los atacantes saturan servidores DNS con tráfico, haciendo dominios inaccesibles (ej., ataque a Dyn en 2016 afectó Twitter, Reddit, Netflix).

DNS amplification attacks exploit open resolvers to multiply traffic volume. Mitigations include rate limiting, anycast distribution, using multiple DNS providers, and response rate limiting (RRL).Los ataques de amplificación DNS explotan resolvers abiertos para multiplicar el volumen de tráfico. Las mitigaciones incluyen limitación de tasa, distribución anycast, usar múltiples proveedores DNS y limitación de tasa de respuesta (RRL).

DNS Hijacking & SpoofingSecuestro y Suplantación DNS

Attackers redirect DNS queries to malicious servers, intercepting traffic or serving fake websites.Los atacantes redirigen consultas DNS a servidores maliciosos, interceptando tráfico o sirviendo sitios falsos.

Cache poisoning (Kaminsky attack) injects false records into resolver caches. DNS hijacking can occur at the registrar level (compromised accounts) or network level (BGP hijacking). DNSSEC and DNS-over-HTTPS help mitigate these threats.El envenenamiento de caché (ataque Kaminsky) inyecta registros falsos en cachés de resolvers. El secuestro DNS puede ocurrir a nivel de registrador (cuentas comprometidas) o red (secuestro BGP). DNSSEC y DNS-over-HTTPS ayudan a mitigar estas amenazas.

SummaryResumen

DNS is Critical InfrastructureDNS es Infraestructura Crítica

DNS translates domain names to IP addresses through a distributed hierarchical system of root, TLD, and authoritative servers. It handles billions of queries daily and is essential for every internet interaction.DNS traduce nombres de dominio a direcciones IP a través de un sistema jerárquico distribuido de servidores raíz, TLD y autoritativos. Maneja miles de millones de consultas diarias y es esencial para cada interacción en internet.

Caching Reduces LatencyEl Caché Reduce Latencia

Multiple caching layers (browser, OS, resolver) minimize DNS lookup times. TTL values control cache duration and must be tuned for your use case — lower for fast failover, higher for reduced query volume.Múltiples capas de caché (navegador, SO, resolver) minimizan tiempos de búsqueda DNS. Los valores TTL controlan la duración del caché y deben ajustarse según el caso — menores para failover rápido, mayores para reducir consultas.

Security is Non-NegotiableLa Seguridad No Es Negociable

DNSSEC, DNS-over-HTTPS, and multi-provider redundancy protect against spoofing, hijacking, and DDoS attacks. Cloud DNS services provide enterprise-grade security and availability out of the box.DNSSEC, DNS-over-HTTPS y redundancia multi-proveedor protegen contra suplantación, secuestro y ataques DDoS. Los servicios DNS en la nube proporcionan seguridad y disponibilidad de nivel empresarial de forma inmediata.
