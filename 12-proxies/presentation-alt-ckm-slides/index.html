What is a Proxy?¿Qué es un Proxy?

A proxy is an intermediary server that sits between clients and backend servers. It intercepts requests and can modify, filter, or route them.Un proxy es un servidor intermediario entre los clientes y los servidores backend. Intercepta solicitudes y puede modificarlas, filtrarlas o enrutarlas.

Forward vs Reverse ProxyProxy Directo vs Proxy Inverso

A forward proxy acts on behalf of clients; a reverse proxy acts on behalf of servers.Un proxy directo actúa en nombre de los clientes; un proxy inverso actúa en nombre de los servidores.

Forward proxies hide client identity from the server (e.g., VPN, corporate firewalls). Reverse proxies hide server identity from the client (e.g., load balancers, CDNs). Both improve security, performance, and control over network traffic.Los proxies directos ocultan la identidad del cliente al servidor (ej., VPN, firewalls corporativos). Los proxies inversos ocultan la identidad del servidor al cliente (ej., balanceadores de carga, CDNs). Ambos mejoran la seguridad, rendimiento y control del tráfico de red.

Forward ProxyProxy Directo

A forward proxy sits in front of clients and forwards their requests to the internet. The server only sees the proxy, not the client.Un proxy directo se coloca frente a los clientes y reenvía sus solicitudes a internet. El servidor solo ve al proxy, no al cliente.

When to UseCuándo Usar

VPN access, content filtering, cachingAcceso VPN, filtrado de contenido, caché

Forward proxies let organizations enforce browsing policies, cache frequently accessed content to save bandwidth, and provide anonymity by masking client IP addresses from destination servers.Los proxies directos permiten a las organizaciones aplicar políticas de navegación, almacenar en caché contenido frecuente para ahorrar ancho de banda y proporcionar anonimato al enmascarar las IPs de los clientes.

ExamplesEjemplos

Corporate networks, schools, Squid proxyRedes corporativas, escuelas, Squid proxy

Corporate firewalls use forward proxies to control which websites employees can access. Schools filter inappropriate content. Squid is a popular open-source forward proxy with caching capabilities.Los firewalls corporativos usan proxies directos para controlar qué sitios web pueden acceder los empleados. Las escuelas filtran contenido inapropiado. Squid es un proxy directo de código abierto con capacidades de caché.

Reverse ProxyProxy Inverso

A reverse proxy sits in front of backend servers and forwards client requests to the appropriate server. The client only sees the proxy.Un proxy inverso se coloca frente a los servidores backend y reenvía las solicitudes de los clientes al servidor apropiado. El cliente solo ve al proxy.

When to UseCuándo Usar

Load balancing, SSL termination, compressionBalanceo de carga, terminación SSL, compresión

Reverse proxies distribute incoming traffic across multiple servers, handle SSL/TLS encryption centrally, compress responses to save bandwidth, and protect backends from direct internet exposure.Los proxies inversos distribuyen el tráfico entrante entre múltiples servidores, manejan el cifrado SSL/TLS de forma centralizada, comprimen respuestas para ahorrar ancho de banda y protegen los backends de la exposición directa a internet.

ExamplesEjemplos

Nginx, HAProxy, Cloudflare, EnvoyNginx, HAProxy, Cloudflare, Envoy

Nginx and HAProxy are the most popular open-source reverse proxies. Cloudflare operates a global reverse proxy network (CDN + DDoS protection). Envoy is a modern proxy designed for service mesh architectures.Nginx y HAProxy son los proxies inversos de código abierto más populares. Cloudflare opera una red global de proxies inversos (CDN + protección DDoS). Envoy es un proxy moderno diseñado para arquitecturas de service mesh.

What is an API Gateway?¿Qué es un API Gateway?

An API Gateway is a central entry point for all API requests. It handles cross-cutting concerns like authentication, rate limiting, and routing so individual services do not have to.Un API Gateway es un punto de entrada central para todas las solicitudes de API. Maneja aspectos transversales como autenticación, limitación de tasa y enrutamiento para que los servicios individuales no tengan que hacerlo.

API Gateway FeaturesFunciones del API Gateway

Request RoutingEnrutamiento

Route requests to the correct backend serviceEnrutar solicitudes al servicio backend correcto

Path-based routing (/users, /orders), header-based routing, query parameter routing, and weighted routing for canary deployments. The gateway decouples clients from backend topology changes.Enrutamiento por ruta (/users, /orders), por encabezados, por parámetros de consulta y enrutamiento ponderado para despliegues canary. El gateway desacopla a los clientes de los cambios de topología del backend.

AuthenticationAutenticación

Verify identity before reaching servicesVerificar identidad antes de llegar a los servicios

JWT validation, OAuth 2.0 token introspection, API key verification, and mTLS certificate validation. Centralizes auth logic so individual services can focus on business logic.Validación de JWT, introspección de tokens OAuth 2.0, verificación de API keys y validación de certificados mTLS. Centraliza la lógica de autenticación para que los servicios se enfoquen en la lógica de negocio.

Rate LimitingLímite de Tasa

Control request volume per clientControlar volumen de solicitudes por cliente

Protects backends from abuse and traffic spikes. Can be configured per API key, IP address, or user identity. Common algorithms include token bucket and sliding window counters.Protege los backends de abuso y picos de tráfico. Se puede configurar por API key, dirección IP o identidad de usuario. Los algoritmos comunes incluyen token bucket y ventana deslizante.

CachingCaché

Cache responses to reduce backend loadCachear respuestas para reducir carga del backend

The gateway can cache GET responses using TTL-based expiration. Reduces latency for frequently requested data and shields backends from redundant queries during traffic spikes.El gateway puede cachear respuestas GET usando expiración basada en TTL. Reduce la latencia para datos solicitados frecuentemente y protege los backends de consultas redundantes durante picos.

Protocol TransformTransformación

Translate between protocols and formatsTraducir entre protocolos y formatos

Convert REST to gRPC, XML to JSON, or aggregate multiple backend calls into a single client response. Enables backend evolution without breaking existing clients.Convertir REST a gRPC, XML a JSON, o agregar múltiples llamadas al backend en una sola respuesta al cliente. Permite la evolución del backend sin romper clientes existentes.

Logging & MonitoringLogs y Monitoreo

Centralized observability for all API trafficObservabilidad centralizada para todo el tráfico API

Request/response logging, latency metrics, error rate tracking, and distributed tracing. A single point to monitor API health across all services without instrumenting each one individually.Registro de solicitudes/respuestas, métricas de latencia, seguimiento de tasas de error y trazado distribuido. Un único punto para monitorear la salud de las APIs sin instrumentar cada servicio individualmente.

Authentication FlowFlujo de Autenticación

The API Gateway centralizes authentication so backend services trust every request that reaches them.El API Gateway centraliza la autenticación para que los servicios backend confíen en cada solicitud que les llega.

Rate LimitingLimitación de Tasa

Rate limiting protects backend services from being overwhelmed by controlling the number of requests a client can make in a given time window.La limitación de tasa protege los servicios backend de ser abrumados al controlar el número de solicitudes que un cliente puede hacer en una ventana de tiempo.

Rate Limiting StrategiesEstrategias de Limitación de Tasa

Token bucket, sliding window, fixed windowToken bucket, ventana deslizante, ventana fija

Token Bucket: Tokens are added at a fixed rate. Each request consumes a token. Allows bursts when the bucket is full. Sliding Window: Counts requests in a rolling time window, providing smooth rate limiting without boundary spikes. Fixed Window: Simple counter reset at fixed intervals, but allows bursts at window boundaries.Token Bucket: Los tokens se agregan a una tasa fija. Cada solicitud consume un token. Permite ráfagas cuando el bucket está lleno. Ventana Deslizante: Cuenta solicitudes en una ventana de tiempo móvil, proporcionando limitación suave sin picos en los límites. Ventana Fija: Contador simple que se reinicia en intervalos fijos, pero permite ráfagas en los límites de la ventana.

Circuit Breaker PatternPatrón Circuit Breaker

The circuit breaker prevents cascading failures by stopping requests to a failing service and allowing it time to recover.El circuit breaker previene fallas en cascada al detener solicitudes a un servicio que está fallando, dándole tiempo para recuperarse.

ClosedCerrado

Normal operation: requests pass throughOperación normal: las solicitudes pasan

All requests are forwarded to the backend service. The circuit breaker monitors failure rates. If failures exceed the configured threshold, the circuit trips to Open state.Todas las solicitudes se reenvían al servicio backend. El circuit breaker monitorea la tasa de fallas. Si las fallas superan el umbral configurado, el circuito cambia al estado Abierto.

OpenAbierto

Fail fast: requests are rejected immediatelyFallo rápido: solicitudes rechazadas

No requests reach the failing service. The gateway returns an immediate error or fallback response. After a configured timeout, the circuit transitions to Half-Open to test recovery.Ninguna solicitud llega al servicio que falla. El gateway devuelve un error inmediato o una respuesta de respaldo. Después del timeout configurado, el circuito transiciona a Semi-Abierto para probar la recuperación.

Half-OpenSemi-Abierto

Test: allow limited requests to check recoveryPrueba: permitir solicitudes limitadas

A small number of test requests are sent to the backend. If they succeed, the circuit closes and normal traffic resumes. If they fail, the circuit reopens and the timeout resets.Se envían pocas solicitudes de prueba al backend. Si tienen éxito, el circuito se cierra y el tráfico normal se reanuda. Si fallan, el circuito se reabre y el timeout se reinicia.

API Gateway PlatformsPlataformas de API Gateway

Major cloud providers and open-source projects offer API Gateway solutions with different strengths and trade-offs.Los principales proveedores de nube y proyectos de código abierto ofrecen soluciones de API Gateway con diferentes fortalezas y compromisos.

AWS API Gateway

Fully managed, serverless, pay-per-requestCompletamente gestionado, serverless, pago por solicitud

REST, HTTP, and WebSocket APIs. Native integration with Lambda, IAM, Cognito. Supports canary deployments, request validation, and usage plans with API keys. No infrastructure to manage.APIs REST, HTTP y WebSocket. Integración nativa con Lambda, IAM, Cognito. Soporta despliegues canary, validación de solicitudes y planes de uso con API keys. Sin infraestructura que administrar.

Azure API Management

Enterprise-grade, developer portal, policiesGrado empresarial, portal de desarrolladores

Built-in developer portal for API documentation. Policy engine for transformations, caching, and validation. Supports hybrid and multi-cloud deployments. Strong enterprise governance features.Portal de desarrolladores integrado para documentación de APIs. Motor de políticas para transformaciones, caché y validación. Soporta despliegues híbridos y multi-nube. Fuertes características de gobernanza empresarial.

Google Cloud Endpoints

OpenAPI spec, gRPC support, Cloud Run integrationEspecificación OpenAPI, soporte gRPC, Cloud Run

API management based on OpenAPI specification. Extensible Service Proxy (ESP) for authentication, monitoring, and logging. Native gRPC transcoding. Tight integration with Cloud Run and GKE.Gestión de APIs basada en la especificación OpenAPI. Extensible Service Proxy (ESP) para autenticación, monitoreo y registro. Transcodificación gRPC nativa. Integración con Cloud Run y GKE.

Plugin-based, cloud-agnostic, Kubernetes-nativeBasado en plugins, agnóstico de nube

Built on Nginx/OpenResty. Extensible with Lua plugins for auth, rate limiting, logging, and transformations. Available as open-source, enterprise, or managed service (Konnect). Kubernetes Ingress Controller for native K8s integration.Construido sobre Nginx/OpenResty. Extensible con plugins Lua para autenticación, limitación de tasa, registro y transformaciones. Disponible como código abierto, empresarial o servicio gestionado (Konnect). Controlador de Ingress para integración nativa con K8s.

When to UseCuándo Usar

Multiple clients, independent service teamsMúltiples clientes, equipos de servicio independientes

Use this pattern when you have multiple client types (web, mobile, IoT) that need a unified API, when services need to be deployed and scaled independently, and when cross-cutting concerns like auth and rate limiting should be centralized.Use este patrón cuando tenga múltiples tipos de clientes (web, móvil, IoT) que necesiten una API unificada, cuando los servicios necesiten desplegarse y escalarse independientemente, y cuando aspectos transversales como autenticación y limitación de tasa deban centralizarse.
